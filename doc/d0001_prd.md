### 1.2 용어 정의

| 용어 | 정의 |
|------|------|
| OCR | Optical Character Recognition (광학 문자 인식), 이미지 내 텍스트를 컴퓨터가 읽을 수 있는 텍스트로 변환하는 기술 |
| EasyOCR | JaidedAI에서 개발한 Python 기반 OCR 오픈소스 라이브러리로 80개 이상의 언어 지원 |
| ROI | Region of Interest, 이미지에서 관심 영역 |

| CRAFT | Character Region Awareness For Text detection, 텍스트 영역 감지 알고리즘 |
| CRNN | Convolutional Recurrent Neural Network, 텍스트 인식에 사용되는 딥러닝 모델 |
| 신뢰도 | Confidence Score, OCR 인식 결과의 정확도 추정치 (0~1) |
| 전처리 | Preprocessing, OCR 성능 향상을 위한 이미지 변환 작업 (이진화, 노이즈 제거 등) |

## 2. 프로젝트 개요

### 2.1 목적

본 프로젝트의 주요 목적은 다음과 같습니다.

1. 제품 라벨 이미지에서 텍스트 자동 인식 시스템 구축
2. EasyOCR 라이브러리의 성능 및 정확도 검증
3. 한국어/영어 혼합 라벨 처리 가능성 검토
4. 실무 적용 가능성 평가 및 최적화 방안 도출

### 2.2 배경

#### 문제 상황

- 제품 라벨 정보를 수작업으로 확인하는 과정이 비효율적
- 대량의 제품 데이터를 처리할 때 시간과 인력 소요 증가
- 라벨 정보 입력 시 인적 오류 발생 가능성

#### 기술적 배경

- OCR 기술의 발전으로 높은 정확도의 자동화 가능
- EasyOCR은 CRAFT 기반 텍스트 감지와 CRNN 기반 인식으로 우수한 성능 제공
- 한국어, 영어 등 다국어 동시 지원으로 혼합 언어 라벨 처리 가능
- Python 생태계와의 높은 호환성

### 2.3 범위

#### In-Scope (범위 내)

- 정적 이미지 파일(JPG, PNG) 기반 제품 라벨 OCR 처리
- 한국어, 영어 텍스트 인식
- 텍스트 추출 및 구조화된 결과 출력
- 인식 결과의 신뢰도 평가
- 바운딩박스 기반 시각화
- 배치 처리 기능 (다중 이미지)
- 인식 결과 파일 저장 (JSON)

#### Out-of-Scope (범위 외)

- 실시간 비디오 스트림 처리
- 대규모 프로덕션 환경 배포
- 바코드/QR코드 인식 (OCR과 별개의 기술)
- 제품 라벨 데이터베이스 구축
- 웹 서비스화
- 모바일 앱 개발
- 자연어 처리 기반 정보 추출 (제품명, 성분 파싱 등)

## 3. 알고리즘 요구사항

### 3.1 입력/출력 정의

#### 입력

| 항목 | 설명 | 형식 | 제약사항 |
|------|------|------|----------|
| 이미지 파일 | 제품 라벨 이미지 | JPG, PNG | 최대 10MB, 최소 해상도 640x480 |
| 언어 설정 | 인식 대상 언어 | ['ko', 'en'] | 기본값: ['ko', 'en'] |
| 전처리 옵션 | 이미지 전처리 여부 | Boolean | 기본값: True |

#### 출력

| 항목 | 설명 | 형식 | 내용 |
|------|------|------|------|
| 텍스트 목록 | 인식된 텍스트 | List[str] | 감지된 각 텍스트 블록의 문자열 |
| 신뢰도 | 인식 정확도 추정치 | List[float] | 0.0 ~ 1.0 범위 |
| 바운딩박스 | 텍스트 영역 좌표 | List[List[Tuple]] | [(x1,y1), (x2,y2), (x3,y3), (x4,y4)] 형식 |
| 처리 시간 | 소요 시간 | float | 초 단위 |

#### 출력 예시

```python
[
    {
        "text": "유통기한: 2026.12.31",
        "confidence": 0.92,
        "bbox": [[10, 20], [200, 20], [200, 50], [10, 50]],
        "language": "ko"
    },
    {
        "text": "Product Name",
        "confidence": 0.88,
        "bbox": [[10, 60], [180, 60], [180, 90], [10, 90]],
        "language": "en"
    }
]
```

### 3.2 핵심 알고리즘

#### 3.2.1 텍스트 감지 (Text Detection)

- 알고리즘: CRAFT (Character Region Awareness For Text detection)
- 목적: 이미지에서 텍스트가 있는 영역 감지
- 특징: 문자 단위 감지로 회전/왜곡된 텍스트도 처리 가능
- 출력: 텍스트 영역의 바운딩박스 좌표

#### 3.2.2 텍스트 인식 (Text Recognition)

- 알고리즘: CRNN (Convolutional Recurrent Neural Network)
- 목적: 감지된 텍스트 영역의 문자열 인식
- 특징: CNN으로 이미지 특징 추출 후 RNN으로 시퀀스 예측
- 출력: 인식된 문자열 및 신뢰도

#### 3.2.3 이미지 전처리

- 목적: OCR 성능 향상을 위한 이미지 품질 개선
- 사용 라이브러리: OpenCV (cv2)
- 주요 기법:
  - 그레이스케일 변환
  - 이진화 (Thresholding)
  - 노이즈 제거 (Denoising)
  - 대비 강화 (Contrast Enhancement)
  - 리사이징 (적절한 해상도 조정)

### 3.3 처리 파이프라인

```
[시작]
   ↓
1. 이미지 로드
   - 파일 경로 검증
   - 이미지 읽기 (Pillow/OpenCV)
   ↓
2. 전처리
   - 그레이스케일 변환
   - 노이즈 제거 (Gaussian Blur)
   - 이진화 (Adaptive Threshold)
   - 대비 강화 (CLAHE)
   - 리사이징 (장변 기준 1280px)
   ↓
3. OCR 실행
   - EasyOCR Reader 초기화
   - 텍스트 감지 (CRAFT)
   - 텍스트 인식 (CRNN)
   ↓
4. 결과 정제
   - 신뢰도 필터링 (threshold < 0.3 제거)
   - 중복 영역 제거
   - 좌표 정규화
   ↓
5. 구조화된 출력
   - JSON/CSV 형식 변환
   - 바운딩박스 시각화 (선택)
   - 결과 파일 저장
   ↓
[종료]
```

#### 파이프라인 상세 설명

**1단계: 이미지 로드**
- 파일 존재 여부 및 형식 검증
- Pillow 또는 OpenCV로 이미지 읽기
- 예외 처리 (파일 손상, 지원하지 않는 형식)

**2단계: 전처리**
- 그레이스케일: 컬러 정보 제거로 처리 속도 향상
- 노이즈 제거: Gaussian Blur로 잡음 감소
- 이진화: Adaptive Threshold로 텍스트/배경 분리
- 대비 강화: CLAHE로 저조도 이미지 개선
- 리사이징: 너무 큰 이미지는 성능 저하 방지를 위해 축소

**3단계: OCR 실행**
- EasyOCR Reader 객체 생성 (언어 설정)
- readtext() 메서드 호출
- GPU 사용 여부 자동 감지

**4단계: 결과 정제**
- 신뢰도 0.3 미만 결과 필터링
- 중복 바운딩박스 IoU 기반 제거
- 좌표를 원본 이미지 크기 기준으로 정규화

**5단계: 구조화된 출력**
- 딕셔너리/리스트 형태로 결과 정리
- 필요 시 시각화 이미지 생성
- 파일 저장 (JSON)

### 3.4 성능 최적화 전략

| 항목 | 전략 | 기대 효과 |
|------|------|----------|
| GPU 활용 | CUDA 지원 시 GPU 사용 | 처리 속도 3~5배 향상 |
| 배치 처리 | 다중 이미지 동시 처리 | I/O 오버헤드 감소 |
| 모델 캐싱 | Reader 객체 재사용 | 초기화 시간 절약 |
| 이미지 리사이징 | 과도한 해상도 축소 | 메모리 사용량 감소 |

### 8.3 평가 지표

#### 8.3.1 정확도 평가

```python
# 문자 단위 정확도 (Character Accuracy)
def char_accuracy(ground_truth, predicted):
    correct_chars = sum(1 for g, p in zip(ground_truth, predicted) if g == p)
    return correct_chars / len(ground_truth)

# 단어 단위 정확도 (Word Accuracy)
def word_accuracy(ground_truth, predicted):
    gt_words = ground_truth.split()
    pred_words = predicted.split()
    correct_words = sum(1 for g, p in zip(gt_words, pred_words) if g == p)
    return correct_words / len(gt_words)

# 레벤슈타인 거리 기반 유사도
def levenshtein_accuracy(ground_truth, predicted):
    distance = levenshtein_distance(ground_truth, predicted)
    max_len = max(len(ground_truth), len(predicted))
    return 1 - (distance / max_len)
```

#### 8.3.2 성능 지표

- Precision: 인식된 텍스트 중 정확한 비율
- Recall: 실제 텍스트 중 인식된 비율
- F1-Score: Precision과 Recall의 조화평균
- 처리 시간: 평균, 중앙값, 95 percentile

## 11. 부록

### 11.1 FAQ

**Q1. EasyOCR과 Tesseract의 차이는?**
- EasyOCR: 딥러닝 기반, 80+ 언어 지원, 설치 간편, GPU 가속 지원
- Tesseract: 전통적 OCR, 100+ 언어 지원, 설정 복잡, CPU 위주

**Q2. GPU가 없어도 사용 가능한가?**
- 예. CPU만으로도 동작하나 처리 속도가 느릴 수 있음 (3~5배 차이)

**Q3. 한글 인식 정확도를 높이려면?**
- 고해상도 이미지 사용
- 적절한 이미지 전처리 (이진화, 대비 조정)
- 신뢰도 낮은 결과 필터링
- 필요 시 후처리 로직 추가 (맞춤법 검사 등)

**Q4. 지원하는 이미지 형식은?**
- JPG, JPEG, PNG (권장)
- BMP, TIFF (가능하나 변환 권장)

**Q5. 상업적 사용이 가능한가?**
- EasyOCR은 Apache 2.0 라이선스로 상업적 사용 가능
- 단, 의존성 라이브러리의 라이선스 별도 확인 필요
