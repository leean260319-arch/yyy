## 1. 개요

이 문서는 `uv` 도구를 활용하여 프로젝트의 Python 의존성을 체계적으로 관리하는 워크플로우를 제공합니다.
`oaischeck` 스킬에서 분리되어 의존성 관리에 특화된 기능을 제공하며, 프로젝트의 안정성과 보안을 유지하는 데 기여합니다.

**핵심 기능:**

- 의존성 상태 확인 (최신 버전 여부, 보안 취약점)
- 의존성 업데이트 제안 및 실행
- `uv run`을 통한 Python 스크립트 실행 지침

## 2. UV 기반 의존성 관리

### 2.1 의존성 체크

`uv`를 사용하여 프로젝트의 `pyproject.toml` 및 `uv.lock` 파일을 기반으로 의존성 상태를 확인합니다.

- **오래된 패키지 감지**: 설치된 패키지 중 최신 버전이 있는지 확인합니다.
- **보안 취약점 감지**: 알려진 보안 취약점이 있는 패키지를 식별합니다.

### 2.2 의존성 업데이트

오래되거나 취약점이 있는 패키지에 대한 업데이트를 제안하고, 사용자 승인 하에 업데이트를 실행합니다.

- **제안 모드**: 업데이트 가능한 패키지 목록과 권장 버전을 표시합니다.
- **자동 업데이트**: `uv update` 명령을 사용하여 `uv.lock` 파일을 업데이트하고, 필요한 경우 패키지를 설치합니다.

### 2.3 Python 실행 지침: `uv run`

프로젝트 내의 Python 스크립트(`ps*.py` 등)는 항상 `uv run` 명령을 사용하여 실행하도록 권장합니다.
이는 `uv`가 관리하는 가상 환경 내에서 스크립트가 실행되도록 보장하여, 의존성 불일치 문제를 방지하고 일관된 실행 환경을 제공합니다.

```bash

# 예시: main.py 스크립트 실행
uv run python src/main.py
```

## 3. 서브명령어 (oaisuv)

`oaisuv` 명령어는 `uv` 기반 의존성 관리를 수행합니다.

| 명령어          | 설명                                                | 관련 키워드          |
|---------------|-----------------------------------------------------|----------------------|
| `oaisuv version` | **스킬 버전 정보 (v02)**                           | `버전`, `version` |
| `oaisuv check` | **의존성 상태 체크 (오래된/취약점 패키지)**         | `업데이트`, `의존성`, `패키지`, `취약점` |
| `oaisuv update` | **오래된/취약점 패키지 업데이트 (실행)**           | `업데이트`, `패치`, `uv update` |
| `oaisuv run`    | **`uv run`을 사용하여 파이썬 스크립트 실행**       | `uv run`, `실행`, `파이썬`   |

### 3.1 사용 예시

| 명령                  | 동작                                        |
|-----------------------|---------------------------------------------|
| `oaisuv check`        | 전체 의존성 상태 확인 (오래된/취약점)         |
| `oaisuv update`       | 오래된 의존성 업데이트 실행                   |
| `oaisuv run python src/main.py` | `uv run`을 통해 `src/main.py` 실행 |

## 4. 통합 및 워크플로우

`oaisuv` 스킬은 `oaischeck` 스킬과 연동하여 프로젝트의 전체적인 코드 품질 및 안정성 관리에 기여합니다.
`oaischeck`가 코드 구문 및 로직 오류를 검사하는 동안, `oaisuv`는 의존성 레벨의 건강 상태를 관리합니다.

```
oaischeck 실행 시:
  - (선택적) oaisuv check를 호출하여 의존성 상태를 확인
  - 문제가 발견되면 doc/d0004_todo.md에 기록
```

## 5. 결과 기록 형식

검출된 의존성 문제는 `doc/d0004_todo.md`에 다음 형식으로 기록:

```markdown

#### TODO-XXX [DEP] 의존성 문제 제목
- **우선순위**: 높음/중간/낮음
- **관련 패키지**: `패키지명`
- **문제**: 문제 설명 (예: 오래된 버전, 보안 취약점)
- **영향**: 프로젝트에 미치는 영향 설명
- **조치**: 권장 해결 방안 (예: `uv update <package_name>`)
- **등록일**: YYYY-MM-DD
```
