## 1. 서브명령어

| 명령어 | 설명 |
|--------|------|
| `oaiscommit version` | 스킬 버전 정보 (v02) |
| `run` | **커밋 + 이력 정리 통합 (기본)** - `--dry-run` 권장 |
| `commit` | Git 커밋만 |
| `sync` | 이력 정리만 (d{SP}0004 → d{SP}0010) |
| `github` | **태스크 → GitHub Issues 변환** (MCP 연동) |
| `status` / `preview` | 서브명령어 리스트, 상태/변경사항 |

## 2. 워크플로우 (5단계)

1. **분석** → git status/diff, 커밋 메시지 생성
2. **커밋** → 스테이징, 실행 (사용자 확인)
3. **추출** → d{SP}0004에서 `[x]`/`해결` 항목 식별
4. **이동** → d{SP}0010 추가, d{SP}0004 제거, 버전 갱신
5. **검증** → 무결성 확인

## 3. 커밋 형식

`<type>(<scope>): <subject>` (Conventional Commits)

| 타입 | 설명 | 타입 | 설명 |
|------|------|------|------|
| `feat` | 새 기능 | `refactor` | 리팩토링 |
| `fix` | 버그 수정 | `test` | 테스트 |
| `docs` | 문서 | `chore` | 기타 |

## 4. 이력 이동 규칙

- **대상**: `[x]`, `해결`, `완료`, `CLOSED` 키워드 항목
- **태그 매핑**: CRITICAL→HOTFIX, ERROR→BUGFIX, WARNING→IMPROVE, INFO→ENHANCE
- **형식**: `### [YYYY-MM-DD] [태그] 제목` + 파일/내용 기술

## 5. GitHub Issues 워크플로우 (github)

**목적**: 태스크(d{SP}0002, tasks.md)를 GitHub Issues로 변환하여 협업 및 추적 용이화

**사용법**:
```bash
oaiscommit github                    # 전체 태스크 변환
oaiscommit github --task-id 1,2,3    # 특정 태스크만
oaiscommit github --dry-run          # 미리보기
oaiscommit github --sync             # 기존 이슈와 동기화
```

**워크플로우**:
```
oaiscommit github
    │
    ├─► 1. 태스크 로드
    │      ├─ d{SP}0002_plan.md (개발 태스크)
    │      ├─ tasks.md (TaskMaster 사용 시)
    │      └─ d{SP}0003_test.md (테스트 태스크)
    │
    ├─► 2. 의존성 분석
    │      ├─ 태스크 간 의존관계 파악
    │      └─ 생성 순서 결정 (의존성 순)
    │
    ├─► 3. GitHub MCP 연동
    │      ├─ mcp__github__create_issue 호출
    │      ├─ 라벨 자동 매핑
    │      └─ 마일스톤 연결 (있는 경우)
    │
    ├─► 4. 이슈 생성
    │      ├─ 제목: 태스크 제목
    │      ├─ 본문: 상세 설명 + 체크리스트
    │      └─ 라벨: priority, type, status
    │
    └─► 5. 결과 반영
           ├─ 이슈 번호 → 태스크 문서에 기록
           └─ 생성 리포트 출력
```

**라벨 매핑**:

| 태스크 속성 | GitHub 라벨 |
|------------|-------------|
| priority: high | `P0-critical` |
| priority: medium | `P1-important` |
| priority: low | `P2-normal` |
| type: feature | `enhancement` |
| type: bug | `bug` |
| type: docs | `documentation` |

**이슈 본문 형식**:
```markdown
## 설명
{태스크 상세 설명}

## 체크리스트
- [ ] 구현 완료
- [ ] 테스트 작성
- [ ] 문서 업데이트

## 관련 정보
- **우선순위**: {priority}
- **의존성**: #{dep_issue_number}
- **원본**: d{SP}0002_plan.md

---
Generated by oaiscommit github
```

**옵션**:

| 옵션 | 설명 |
|------|------|
| `--task-id <ids>` | 특정 태스크만 변환 (쉼표 구분) |
| `--dry-run` | 실제 생성 없이 미리보기 |
| `--sync` | 기존 이슈와 양방향 동기화 |
| `--labels <labels>` | 추가 라벨 지정 |
| `--milestone <name>` | 마일스톤 지정 |
| `--project <name>` | GitHub Project 연결 |

## 6. 일반 옵션

| 옵션 | 설명 |
|------|------|
| `--message "msg"` | 커밋 메시지 지정 |
| `--no-push` | push 생략 |
| `--dry-run` | 미리보기 |
| `--force-sync` | 완료 항목 강제 이동 |

## 7. 주의사항

- 명시적 요청 없이 자동 커밋 금지 (확인 필수)
- 테스트/린트 통과, 민감정보 제외
- 완료 상태 명확한 항목만 이동
