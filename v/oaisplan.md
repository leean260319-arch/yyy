## 5. 워크플로우

### 5.1 run
PRD 로드 → Epic 도출 → Feature 분리 → Task 세분화 → d{SP}0002_plan.md 저장

**PRD 필요 조건** (웹 개발 프로젝트):

| 옵션 | PRD 필요 섹션 | 근거 |
|------|-------------|------|
| `run epic` | 페이지 개요 (5.1) | 대분류 기반 Epic 도출 |
| `run feature` | **페이지 상세 (5.2)** | 페이지별 기능 정의 필요 |
| `run task` | 페이지 상세 (5.2) + 상세기획 | 구현 단위 분해 |

> **Note**: 웹 프로젝트에서 Feature는 페이지별 기능이므로 5.2 페이지 상세의 "주요 기능" 컬럼이 필수

**진행 컬럼 필터링**: PRD 5.1 페이지 개요의 `진행=아니오`인 페이지는 Plan 생성 시 자동 제외

**하위 레벨 처리**:

| 옵션 | 범위 | 기존 하위 레벨 |
|------|------|--------------|
| `run epic` | Epic까지 | Feature/Task 삭제 |
| `run feature` | Feature까지 | Task 삭제 |
| `run task` (기본) | Task까지 | 전체 업데이트 |

### 5.2 detail
입력 분석 → 태스크 구체화 → 실행 계획 → 사용자 승인

### 5.3 optimize
Plan 로드 → PRD 정합성 → 구조/일정 최적화 → 리스크 검토 → 업데이트

**체크**: PRD 커버리지 100% / 순환 의존성 없음 / 크리티컬 패스 최소화 / 스프린트 균형

### 5.4 sync
PRD 버전 확인 → 변경 분석 → Plan 업데이트

## 6. PRD → Plan 매핑

### 6.1 계층 매핑
PRD 비전→목표요약 / 기능→Epic / 아키텍처→결정사항 / 마일스톤→스프린트 / 리스크→상세화

### 6.2 분해 체계
```
PRD → Epic(도메인) → Feature(기능) → Task(구현) → TC(테스트)
```

| 레벨 | 기준 | 예시 |
|------|------|------|
| Epic | 도메인/비즈니스 | 정책정보, 인증 |
| Feature | 사용자 기능 | 중앙정부 정책 |
| Task | 구현 단위 | 보건복지부 탭 |

## 10. 딥러닝 프로젝트 실험 결과 표기

### 10.1 개요

딥러닝/ML 프로젝트에서 실험 결과를 체계적으로 문서화하기 위한 표기 가이드.

### 10.2 실험 매트릭스 형식

Phase별 실험 결과는 아래 형식의 테이블로 정리:

```markdown
#### X.X.X 실험 매트릭스 (검증된 [주요 메트릭] 기준)

| # | 실험 ID | 변경 사항 | 하이퍼파라미터1 | 하이퍼파라미터2 | 상태 | 주요메트릭 |
|---|---------|-----------|----------------|----------------|------|-----------|
| 1 | ps31XX | 베이스라인 | 1e-4 | - | 완료 | 0.9500 |
| 2 | ps31XX | 변형1 | 1e-5 | 10 | 완료 | 0.9550 |
```

### 10.3 필수 컬럼

| 컬럼 | 설명 | 예시 |
|------|------|------|
| # | 실험 순번 | 1, 2, 3... |
| 실험 ID | 스크립트 파일명 (버전 제외) | ps3110, ps3120 |
| 변경 사항 | 베이스라인 대비 변경점 | "Warmup 10 에폭 추가" |
| 하이퍼파라미터 | 주요 변경 파라미터들 | LR, Warmup, Patience 등 |
| 상태 | 실험 상태 | 완료, 진행중, 실패 |
| 주요 메트릭 | 검증용 핵심 지표 | val_iou, val_loss 등 |

### 10.4 종합 비교표 형식

Phase 완료 시 전체 실험을 순위별로 정리:

```markdown
#### X.X 실험 결과 종합

| 순위 | 모델 | 스크립트 | 파라미터 수 | 주요메트릭 | 베이스라인 대비 |
|------|------|----------|------------|-----------|---------------|
| **1위** | **최고 성능 모델** | **psXXXX** | XXM | **0.9800** | **+X.XX%p** |
| 2위 | 2순위 모델 | psXXXX | XXM | 0.9750 | +X.XX%p |
| 3위 | 베이스라인 | psXXXX | XXM | 0.9700 | - |
```

### 10.5 메트릭 검증 원칙

1. **원본 데이터 기준**: 체크포인트 파일명, 로그, 또는 결과 CSV에서 추출
2. **검증 스크립트 활용**: `tmp/extract_metrics.py` 등으로 자동 검증
3. **불일치 시 원본 우선**: 문서와 실제 데이터 불일치 시 원본 데이터로 수정
4. **무효 실험 제외**: 학습 중단, 결과 파일 없음 등은 표에서 제외

### 10.6 상태 표기

| 상태 | 의미 | 조건 |
|------|------|------|
| 완료 | 학습+테스트 완료 | 체크포인트 + Excel + CSV 존재 |
| 진행중 | 학습 진행 중 | 체크포인트만 존재 |
| 실패 | 학습 중단/오류 | 불완전한 결과 |
| 대기 | 실험 예정 | 스크립트만 존재 |
