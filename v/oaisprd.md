## 1. 개요

PRD 생성 및 코드/문서 정합성 검증. 서브에이전트(Explore, task-checker) 병렬처리 지원.

- **컨텍스트**: `--sp N` 또는 `oaiscontext N`
- **출력**: `doc/d{SP}0001_prd.md`
- **연계**: 이슈→`d{SP}0004_todo.md`, 개발→`d{SP}0002_plan.md`
- **템플릿**: `v/template/prd/` (프로젝트 유형별)

> **Note**: 테스트 문서(`d0003_test.md`)는 `oaisdev` 첫 실행 시 자동 생성

## 2. 서브명령어

| 명령어 | 설명 | 출력 |
|--------|------|------|
| `oaisprd status` | 서브명령어 리스트, 상태 요약 | 터미널 |
| `oaisprd version` | 스킬 버전 정보 (v09) | 터미널 |
| `oaisprd run` | PRD 생성/정합성 검증 | d{SP}0001_prd.md |
| `oaisprd run --template [type]` | 템플릿 지정 생성 | d{SP}0001_prd.md |
| `oaisprd optimize` | PRD 최적화 | d{SP}0001_prd.md |
| `oaisprd template` | 템플릿 목록 조회 | 터미널 |
| `oaisprd template [type]` | 특정 템플릿 조회 | 터미널 |
| `oaisprd validate` | 구조 검증 | 터미널 |
| `oaisprd section [N]` | 특정 섹션 갱신 | d{SP}0001_prd.md |
| `oaisprd unitdev` | 전체 단위개발문서 현행화 | d{SP}1XXX_*.md |
| `oaisprd unitdev [문서명]` | 특정 단위개발문서 현행화 | 해당 문서 |
| `oaisprd clarify` | PRD 모호성 해소 (최대 5개 질문) | d{SP}0001_prd.md |

## 3. 워크플로우

### 3.1 run 분기
`run` → PRD 존재? → 없음: 신규 생성 / 있음: 정합성 검증

### 3.2 신규 생성
프로젝트 정보 수집 → 템플릿 기반 초안 → `d{SP}0001_prd.md` 저장

### 3.3 정합성 검증
PRD 로드 → 비교대상 스캔 → 불일치 → 리포트 → `d{SP}0004_todo.md` 등록

**비교 대상**: 코드베이스, d{SP}0002_plan, d{SP}0003_test, d{SP}0005_lib, d{SP}0006_db, d{SP}0008_user

### 3.4 optimize
PRD 로드 → 구조/내용 검증 → 일관성 최적화

**체크**: 템플릿 완성, 모호함 0건, MoSCoW 할당, 중복 제거

### 3.5 unitdev (단위개발문서 현행화)

> **핵심 원칙**: 앞 섹션(1. 개요)을 기준으로 뒤 섹션(2~5)을 정합성 맞춤

```
1. 개요 ─────► 2. 기획 ─────► 3. 설계 ─────► 4. 구현 ─────► 5. 검증
   │              │              │              │              │
   ▼              ▼              ▼              ▼              ▼
 목적/아이템    아이템→기능    기능→아키텍처   설계→코드     구현→테스트
```

**섹션 연계 규칙**:

| 입력 섹션 | 출력 섹션 | 연계 규칙 |
|----------|----------|----------|
| 1.1 목적 | 2.1 기능요구사항 | 목적 달성을 위한 기능 도출 |
| 1.2 핵심가치 | 5.2 검증체크리스트 | 핵심가치별 검증항목 생성 |
| 1.3 아이템 | 2.1 기능요구사항 | 아이템별 기능 매핑 |
| 1.3.N 아이템상세 | 3.N 데이터구조 | 상세정보 → 필드 정의 |
| 2.1 기능요구사항 | 4.2 주요함수 | 기능별 함수 매핑 |
| 2.2 시나리오 | 5.1 테스트케이스 | 시나리오별 TC 생성 |
| 2.3 UI/UX | 3.1 아키텍처 | UI 구성요소 → 컴포넌트 |
| 3.N 데이터구조 | 4.5 DB | 필드 → 테이블 정의 |

**워크플로우**:
```
oaisprd unitdev [문서명]
    │
    ├─► 1. 문서 로드 (단일 또는 전체)
    │
    ├─► 2. 개요(1장) 분석
    │      ├─ 1.1 목적 추출
    │      ├─ 1.2 핵심가치 추출
    │      └─ 1.3 아이템/상세 추출
    │
    ├─► 3. 섹션별 정합성 검증
    │      ├─ 2장: 아이템 → 기능 매핑 확인
    │      ├─ 3장: 기능 → 아키텍처/데이터구조 확인
    │      ├─ 4장: 설계 → 함수/DB 매핑 확인
    │      └─ 5장: 시나리오 → TC 매핑 확인
    │
    ├─► 4. 불일치 항목 자동 업데이트
    │      ├─ 누락된 기능/함수 추가
    │      ├─ 데이터구조 동기화
    │      └─ 테스트케이스 생성
    │
    └─► 5. 문서 저장 및 버전 업데이트
```

**사용법**:
```bash
# 현재 컨텍스트(SP)의 모든 단위개발문서 현행화
oaisprd unitdev

# 특정 문서만 현행화
oaisprd unitdev d21720
oaisprd unitdev d21720_앱_시니어복지_단위개발.md
```

**출력 예시**:
```
[unitdev] d21720 분석 중...
  - 1.1 목적: 사용자 맞춤형 복지혜택 표출
  - 1.3 아이템: 복지정보카드 (7개 필드)

[정합성 검증]
  - 2.1 기능요구사항: 2건 누락 → 추가
  - 4.2 주요함수: 3건 누락 → 추가
  - 5.1 테스트케이스: 4건 누락 → 생성

[완료] d21720 v05 → v06 업데이트
```

### 3.6 clarify (모호성 해소)

> **목적**: PRD 내 불명확한 영역을 식별하고 최대 5개 질문으로 해소

**검사 영역 (Coverage Taxonomy)**:

| 영역 | 검사 항목 |
|------|----------|
| 기능 범위 | 핵심 목표, 제외 선언, 사용자 역할 |
| 도메인/데이터 | 엔티티, 관계, 상태 전이, 볼륨 |
| UX 흐름 | 핵심 여정, 에러/빈/로딩 상태, 접근성 |
| 비기능 | 성능, 확장성, 보안, 관찰성 |
| 통합 | 외부 서비스, 실패 모드, 프로토콜 |
| 엣지케이스 | 네거티브 시나리오, 충돌 해결 |

**워크플로우**:
```
oaisprd clarify
    │
    ├─► 1. PRD 로드 (d{SP}0001_prd.md)
    │
    ├─► 2. Coverage 스캔
    │      각 영역별 상태: Clear / Partial / Missing
    │
    ├─► 3. 질문 생성 (최대 5개)
    │      ├─ 영향도 × 불확실성 우선순위화
    │      ├─ 다중선택 또는 단답형
    │      └─ 권장 옵션 제시
    │
    ├─► 4. 순차 질문 (1개씩)
    │      ├─ 사용자 응답 대기
    │      └─ "done" 시 조기 종료
    │
    ├─► 5. PRD 업데이트
    │      ├─ ## Clarifications 섹션 추가
    │      └─ 해당 섹션에 명확화 내용 반영
    │
    └─► 6. 완료 리포트
           ├─ 질문/응답 수
           ├─ 수정된 섹션
           └─ 커버리지 요약
```

**질문 형식**:
```markdown
## Question [N]: [주제]

**Context**: [PRD 관련 섹션 인용]

**What we need to know**: [구체적 질문]

**Suggested Answers**:

| Option | Answer | Implications |
|--------|--------|--------------|
| A (권장) | [첫 번째 제안] | [영향] |
| B | [두 번째 제안] | [영향] |
| C | [세 번째 제안] | [영향] |
```

**사용법**:
```bash
oaisprd clarify              # PRD 모호성 해소 시작
oaisprd clarify --max 3      # 최대 3개 질문으로 제한
```

## 4. 정합성 검증 항목

### 코드 vs PRD
| 검증 항목 | PRD 섹션 | 비교 대상 |
|----------|---------|----------|
| 기능 구현 | 4. 기능 요구사항 | 구현 파일/함수 |
| 기술 스택 | 5.2 기술 스택 | pyproject.toml |
| API 엔드포인트 | 5.3 API 명세 | 라우터/컨트롤러 |
| DB 스키마 | 5.4 DB 스키마 | 실제 테이블 |

### 문서 간 비교
| 검증 항목 | PRD 섹션 | 비교 문서 |
|----------|---------|----------|
| 구현 계획 | 4. 기능 요구사항 | d{SP}0002_plan.md |
| 테스트 커버리지 | 7. 테스트 전략 | d{SP}0003_test.md |
| 라이브러리 | 5.2 기술 스택 | d{SP}0005_lib.md |
| DB 구조 | 5.4 DB 스키마 | d{SP}0006_db.md |

## 5. 불일치 리포트

**터미널 출력**: `[불일치: N건]` + 항목별 상세
**등록 형식**: `| ID | 발생일 | 분류 | 내용 | 우선순위 | 상태 |`
**분류**: `[PRD]` 정합성, `[DOC]` 문서 불일치

## 7. PRD 템플릿

### 7.1 템플릿 목록

> 위치: `v/template/prd/`

| 템플릿 | 유형 | 용도 | 예시 |
|--------|------|------|------|
| `common_prd_template.md` | 공통 | 범용 PRD | d0001 |
| `streamlit_prd_template.md` | Type B | Streamlit 웹 앱 | d20001 |
| `algorithm_prd_template.md` | Type A | 알고리즘/분석 | d10001 |
| `agent_prd_template.md` | Type C | 에이전트/CLI | - |

### 7.2 템플릿 선택 규칙

| 프로젝트 유형 | 조건 | 템플릿 |
|--------------|------|--------|
| Type A | 알고리즘/연구/분석 | algorithm |
| Type B | Streamlit 웹 서버 | streamlit |
| Type C | 에이전트/CLI 도구 | agent |
| 기타 | 범용 | common |

### 7.3 템플릿 사용

```bash
# 템플릿 목록 조회
oaisprd template

# 특정 템플릿으로 PRD 생성
oaisprd run --template streamlit
oaisprd run --template algorithm
```

### 7.4 템플릿 공통 목차

1~4: 문서관리, 개요, 사용자/기능 요구사항
5~7: 기술, 비기능, 테스트/품질
8~10: 배포/운영, 관리, 부록

**우선순위**: Must(필수), Should(권장), Could(선택), Won't(제외)

### 7.5 페이지 개요 컬럼 (Streamlit 템플릿)

5.1 페이지 개요 표 필수 컬럼:

| 컬럼 | 설명 |
|------|------|
| ID | 페이지 식별자 |
| 대분류 | 메뉴 그룹 |
| 메뉴명 | 표시명 |
| 설명 | 기능 설명 |
| 권한 | 접근 권한 |
| 진행 | 예/아니오 |
| 기획문서링크 | 상세 기획문서 번호 (d2XXXX) |

> **규칙**: `진행=아니오`인 페이지는 oaisplan/oaisdev에서 자동 제외

## 8. 검증 체크리스트

| 우선순위 | 섹션 | 검증 항목 |
|---------|------|----------|
| Must | 1,2,4,5 | 버전, 목표, 기능목록, 기술스택 |
| Should | 3,6,7 | 페르소나, 성능/보안, 테스트 |
| Could | 8~10 | 배포, 관리, 부록 |
