## 4. 설계

### 4.1 데이터베이스 설계

**테이블명**: `{테이블명}` (신규)

| 컬럼명 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `id` | INTEGER | PK | ID (Auto Increment) |
| `user_id` | TEXT | FK | 작성자 ID (common_user.id) |
| `created_at` | DATETIME | O | 생성일시 |
| `updated_at` | DATETIME | O | 수정일시 |

### 4.2 로직 흐름 (Process Logic)

1. **페이지 초기화 (`initialize_page`)**:
    - 로그인 상태 및 세션 확인
    - 사용자 정보 컴포넌트 렌더링

2. **입력 폼 처리**:
    - `st.form`을 사용하여 일괄 제출 구조 적용
    - 필드별 입력 및 검증

3. **데이터 저장**:
    - Validation 수행
    - DB INSERT/UPDATE
    - Commit/Rollback 처리

4. **완료 처리**:
    - 성공 메시지 표시
    - 다음 동작 안내

### 4.3 디렉토리 및 파일 구조

- **페이지**: `02_1st_server/pages/{N}_{NN}_{페이지명}.py`
- **파일 저장 경로**: `file/{기능명}/` (필요시)
- **DB 파일**: `02_1st_server/db/sene.sqlite`

### 4.4 oais 모듈 활용

#### 4.4.1 사용자 인증/정보 (`oais/__init__.py`)

| 함수명 | 용도 | 사용 위치 |
|--------|------|----------|
| `check_login_status()` | 로그인 상태 확인 | 페이지 초기화 |
| `get_user_info()` | 현재 사용자 정보 조회 | 기본 정보 표시 |
| `get_user_id()` | 사용자 ID 반환 | 데이터 저장 시 |

#### 4.4.2 DB 연결 및 쿼리 (`oais/__init__.py`)

| 함수명 | 용도 | 사용 위치 |
|--------|------|----------|
| `get_db_connection()` | DB 연결 객체 | 전체 DB 작업 |
| `exec_cud_query(sql)` | INSERT/UPDATE 실행 | 데이터 저장 |
| `exec_r_query(sql)` | SELECT 실행 | 데이터 조회 |

#### 4.4.3 날짜/시간 (`oais/__init__.py`)

| 함수명 | 용도 | 사용 위치 |
|--------|------|----------|
| `dt_str_gen()` | 현재 날짜시간 문자열 | created_at, updated_at |
| `get_today()` | 오늘 날짜 (YYYYMMDD) | 파일 저장 경로 생성 |

#### 4.4.4 구현 예시 코드

```python
import oais

# 1. 로그인 확인
if not oais.check_login_status():
    st.warning("로그인이 필요합니다.")
    st.stop()

# 2. 사용자 정보
user_info = oais.get_user_info()
user_id = oais.get_user_id()

# 3. 데이터 조회
sql = "SELECT * FROM {테이블명} WHERE user_id = ?"
data = oais.exec_r_query(sql, [user_id])

# 4. 데이터 저장
insert_sql = f"""
    INSERT INTO {테이블명} (user_id, ..., created_at)
    VALUES ('{user_id}', ..., '{oais.dt_str_gen()}')
"""
oais.exec_cud_query(insert_sql)
```
