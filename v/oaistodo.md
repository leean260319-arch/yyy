## 1. 개요

d{SP}0004_todo.md 문서의 "커스텀 Todo" 섹션을 관리하고, 업무를 자동으로 처리합니다.

**핵심 기능**:
1. **추가+즉시처리**: `oaistodo "내용"` → 추가 후 바로 처리
2. **일괄 처리**: `oaistodo` → 대기 중 업무 전체 처리
3. **추가만**: `oaistodo add "내용"` → 추가만 하고 나중에 처리
4. **상태 조회**: `oaistodo status` → 대기 목록 표시
5. **아카이브**: `oaistodo clear` → 완료 항목을 d0010_history.md로 이동

**옵션**: `--sp N` (서브프로젝트)

## 2. 서브명령어

| 명령어 | 설명 |
|--------|------|
| `oaistodo "내용"` | **추가 + 즉시 처리** (권장) |
| `oaistodo` | 대기 중 업무 전체 처리 |
| `oaistodo status` | 서브명령어 리스트, 대기 목록 표시 |
| `oaistodo version` | 스킬 버전 정보 (v07) |

| `oaistodo clear` | 완료 항목 → d0010_history.md 아카이브 |

실행: `uv run python v/script/oaistodo_run.py [args]`

## 3. 자동 처리 워크플로우

### 3.1 추가+즉시처리 (`oaistodo "내용"`)

```
oaistodo "버그 수정" 실행
    ↓
d{SP}0004_todo.md "커스텀 Todo" 섹션에 추가
    ↓
즉시 처리 지시 출력
    ↓
에이전트가 해당 업무 처리
    ↓
완료 시 "완료" 섹션으로 이동
```

### 3.2 일괄 처리 (`oaistodo`)

```
oaistodo 실행
    ↓
d{SP}0004_todo.md "커스텀 Todo" 섹션 읽기
    ↓
대기 중 항목 없음? → "처리할 업무가 없습니다" 출력 후 종료
    ↓
대기 중 항목 있음:
    ↓
각 항목에 대해:
    1. 항목을 "진행 중"으로 이동
    2. oaisdev run 호출 (자동 처리)
    3. 완료 시 "완료" 섹션으로 이동
    ↓
모든 항목 처리 완료까지 반복
```

### 3.3 추가만 (`oaistodo add`)

`oaistodo add` 명령은 **"커스텀 Todo" 섹션**에 항목만 추가하고 처리하지 않습니다.

```
oaistodo add "내용"
    ↓
d{SP}0004_todo.md "커스텀 Todo" 섹션 확인
    ↓
섹션 없음? → 섹션 자동 생성 (문서 끝)
    ↓
테이블에 항목 추가 (처리는 나중에)
```

> 템플릿: `v/template/common_todo_template.md` 참조

### 3.4 자동 처리 방식

대기 중 업무는 **작업 유형에 따라 분기 처리**됩니다.

```
커스텀 Todo 항목 읽기
    ↓
작업 유형 판별
    ↓
┌─────────────────┬─────────────────┐
│ 코딩 작업       │ 비코딩 작업      │
│ oaisdev run    │ 직접 처리        │
└─────────────────┴─────────────────┘
    ↓
완료 처리
```

### 3.5 작업 유형 판별 룰셋

| 유형 | 키워드/패턴 | 처리 방식 |
|------|------------|----------|
| **코딩** | 구현, 개발, 수정, 버그, 리팩토링, 테스트, API, 함수, 클래스 | `oaisdev run` |
| **비코딩** | 문서, 설명, 작성, 정리, 분석, 검토, 아키텍처, 다이어그램 | 직접 처리 |

### 3.6 비코딩 작업 처리 규칙

비코딩 작업은 다음 방식으로 직접 처리합니다:

| 작업 유형 | 처리 방법 |
|----------|----------|
| 문서 작성/수정 | 해당 문서 읽기 → 내용 추가/수정 → 저장 |
| 설명 추가 | 대상 섹션 확인 → 상세 설명 작성 → 삽입 |
| 분석/검토 | 대상 읽기 → 분석 결과 정리 → 문서화 |
| 정리/구조화 | 현재 상태 확인 → 재구성 → 업데이트 |

> **핵심**: 비코딩 작업은 TDD 프로세스 없이 바로 실행

### 3.7 상태 전이

```
[대기 중] → [진행 중] → [완료]
              ↓ (실패 시)
            [디버깅 섹션에 등록]
```

### 3.8 완료 항목 아카이브 (`oaistodo clear`)

완료된 이슈를 d0010_history.md로 이동하여 d0004_todo.md를 정리합니다.

```
oaistodo clear 실행
    ↓
d{SP}0004_todo.md "해결된 이슈" 섹션 읽기
    ↓
완료 항목 없음? → "아카이브할 이슈가 없습니다" 출력 후 종료
    ↓
완료 항목 있음:
    ↓
d{SP}0010_history.md "아카이브된 이슈" 섹션에 추가
    ↓
d{SP}0004_todo.md "해결된 이슈" 섹션 초기화
    ↓
완료 리포트 출력
```

**권장 주기**: 1~2주마다 실행하여 d0004_todo.md 크기 관리

## 4. 사용 예시

### 추가+즉시처리 (권장)
```bash
# 추가하고 바로 처리
oaistodo "버그 수정"
oaistodo "로그인 API 구현"
oaistodo "문서 업데이트" --priority high
```

### 일괄 처리
```bash
# 대기 중 업무 전체 처리
oaistodo

# 서브프로젝트 지정
oaistodo --sp 02
```

### 상태 조회
```bash
oaistodo status
```

**출력 예시**:
```
### 서브명령어

| 명령어 | 설명 |
|--------|------|
| `oaistodo "내용"` | 추가 + 즉시 처리 (권장) |
| `oaistodo` | 대기 중 업무 전체 처리 |
| `oaistodo status` | 서브명령어 리스트, 상태 조회 |
| `oaistodo add [text]` | 할 일 추가만 (처리 안함) |

### 현재 대기 중 (2건)

| ID | 등록일 | 내용 | 우선순위 | 비고 |
|----|--------|------|---------|------|
| T004 | 2026-01-15 | oaistodo 명령어 생성 | Medium | - |
| T005 | 2026-01-15 | oaissync diff 기능 | High | - |
```

### 할 일 추가만 (나중에 처리)
```bash
# 추가만 (처리 안함)
oaistodo add "새로운 기능 구현"

# 우선순위 지정
oaistodo add "긴급 버그 수정" --priority high

# 비고 추가
oaistodo add "API 문서화" --note "v2.0용"
```

### 완료 항목 아카이브
```bash
# 완료 항목을 d0010_history.md로 이동
oaistodo clear

# 미리보기 (실제 이동 없음)
oaistodo clear --dry-run

# 서브프로젝트 지정
oaistodo clear --sp 02
```

## 5. 옵션

| 옵션 | 설명 | 기본값 |
|------|------|--------|
| `--sp N` | 서브프로젝트 지정 (d{SP}0004 사용) | 00 |
| `--priority [high\|medium\|low]` | 우선순위 지정 (add 시) | medium |
| `--note [text]` | 비고 추가 (add 시) | - |
| `--dry-run` | 실제 처리 없이 계획만 표시 | false |
| `--max-items N` | 최대 처리 항목 수 | 전체 |
