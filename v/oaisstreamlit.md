# oaisstreamlit - Streamlit 페이지 개발 통합 가이드

> 중앙 안내: 상세한 에이전트 활용 원칙 및 공통 가이드라인은 `v/guide/common_guide.md`의 '2. 에이전트 활용 원칙' 섹션을 참조하십시오.

## 개요

**통합 문서 방식**은 하나의 페이지 개발에 필요한 모든 정보(PRD → 플랜 → 설계 → 구현 → 검증)를 **단일 문서**에 담아 개발하는 방식입니다.

### 문서 구조

| 단계 | 내용 | 담당 에이전트 |
|------|------|--------------|
| **Phase 1: 개요** | 개발서 생성 및 기본 개요 작성 | User / Command |
| **Phase 2: PRD** | 페이지 상세 기획, 기능/데이터 정의 | `streamlit-page-planner` |
| **Phase 3: 플랜** | 구현 계획(WBS), 우선순위, 리스크 관리 | `streamlit-page-planner` |
| **Phase 4: 설계** | UI/UX 레이아웃, 컴포넌트, DB 설계 | `streamlit-page-designer` |
| **Phase 5: 구현** | oais 모듈 활용 코드 구현 | `streamlit-implementer` |
| **Phase 6: 검증** | 코드 리뷰, 품질 분석, E2E 테스트 | 다중 에이전트 |

### 관련 에이전트 정의 파일

| 에이전트 | 정의 파일 | 역할 |
|----------|----------|------|
| streamlit-page-planner | `v/agent/streamlit-page-planner.md` | PRD 및 플랜 |
| streamlit-page-designer | `v/agent/streamlit-page-designer.md` | 설계 |
| streamlit-implementer | `v/agent/streamlit-implementer.md` | 구현 |
| python-code-reviewer | `v/agent/python-code-reviewer.md` | 코드 리뷰 |
| oaisqa | `v/agent/oaisqa.md` | 품질 분석 |
| web-test-orchestrator | `v/agent/oais-web-test-orchestrator.md` | E2E 테스트 |
| task-checker | `v/agent/task-checker.md` | 완료 검증 |
| code-error-checker | `v/agent/code-error-checker.md` | 에러 체크 |

# 명령어

## 서브명령어

| 명령어 | 설명 |
|--------|------|
| `oaisstreamlit version` | 스킬 버전 정보 (v01) |
| `oaisstreamlit 생성 [파일경로]` | 템플릿에서 새 개발서 생성 |
| `oaisstreamlit 개요 [파일경로] [내용]` | 개발서의 개요 섹션에 내용 작성 |
| `oaisstreamlit` | 전체 워크플로우 실행 (PRD→플랜→설계→구현→검증) |
| `oaisstreamlit [페이지명]` | 특정 페이지 전체 워크플로우 |
| `oaisstreamlit prd [페이지명]` | Phase 2(PRD) 실행 |
| `oaisstreamlit 플랜 [페이지명]` | Phase 3(플랜) 실행 |
| `oaisstreamlit 설계 [페이지명]` | Phase 4(설계) 실행 |
| `oaisstreamlit 구현 [페이지명]` | Phase 5(구현) 실행 |
| `oaisstreamlit 검증 [페이지명]` | Phase 6(검증) 실행 |
| `oaisstreamlit 리뷰 [파일경로]` | Streamlit 코드 전용 리뷰 실행 |

### 생성 명령어 상세

**용도**: 템플릿(`v/template/streamlit-page.md`)을 복사하여 새 개발서 파일 생성

**문법**:

```bash
oaisstreamlit 생성 doc/d{번호}_{페이지명}.md
```

**예시**:

```bash

# 세무문의요청 페이지 개발서 생성
oaisstreamlit 생성 doc/d22410_세무문의요청.md

# 대리업체관리 페이지 개발서 생성
oaisstreamlit 생성 doc/d22501_대리업체관리.md
```

**동작**:

1. `v/template/streamlit-page.md` 템플릿 읽기
2. 지정한 경로에 파일 복사
3. 생성 완료 메시지 출력

**실행 로직** (Claude가 따라야 할 단계):

```
1. 템플릿 파일 읽기:
   Read("v/template/streamlit-page.md")

2. 대상 파일 생성:
   Write(file_path="[사용자가 지정한 경로]", content=[템플릿 내용])

3. 결과 출력:
   "✅ 개발서 생성 완료: [파일경로]"
   "📝 다음 단계: 섹션 1(개요)부터 작성을 시작하세요."
```

### 개요 명령어 상세

**용도**: 개발서의 "1. 개요" 섹션에 내용을 정리하여 작성

**문법**:

```bash
oaisstreamlit 개요 [파일경로] [개요 내용]
```

**예시**:

```bash

# 세무문의요청 페이지 개요 작성
oaisstreamlit 개요 doc/d22410_세무문의요청.md 세무사에게 문의사항을 요청하고 답변을 받는 페이지

# 대리업체관리 페이지 개요 작성
oaisstreamlit 개요 doc/d22501_대리업체관리.md 대리업체 정보를 등록, 조회, 수정, 삭제하는 관리 페이지
```

**동작**:

1. 지정한 파일 읽기
2. 제공된 내용을 정리하여 "1. 개요" 섹션에 작성
3. 기존 개요 내용이 있으면 교체

**실행 로직** (Claude가 따라야 할 단계):

```
1. 대상 파일 읽기:
   Read("[파일경로]")

2. 개요 내용 구조화:
   - 목적: [한 줄 요약]
   - 대상 사용자: [추정 또는 명시]
   - 주요 기능: [핵심 기능 나열]
   - 비고: [추가 참고사항]

3. "1. 개요" 섹션에 내용 삽입:
   Edit(
     file_path="[파일경로]",
     old_string="## 1. 개요\n\n<!-- 페이지 개요 작성 -->",
     new_string="## 1. 개요\n\n[구조화된 개요 내용]"
   )

4. 결과 출력:
   "✅ 개요 작성 완료: [파일경로]"
   "📝 작성된 내용:"
   [구조화된 개요 요약]
```

**개요 작성 형식**:

```markdown

## 1. 개요

**목적**: [페이지의 핵심 목적]

**대상 사용자**: [사용자 유형]

**주요 기능**:
- [기능 1]
- [기능 2]
- [기능 3]

**비고**: [추가 참고사항, 선택적]
```

### 리뷰 명령어 상세

**용도**: Streamlit 페이지 코드를 전용 리뷰어로 검토하여 Session State, 캐싱, oais 모듈 통합 등 Streamlit 특화 문제를 진단

**문법**:

```bash
oaisstreamlit 리뷰 [파일경로] [옵션]
```

**예시**:

```bash

# 세무문의요청 페이지 리뷰
oaisstreamlit 리뷰 02_1st_server/pages/4_41b_세무서비스문의요청.py

# Session State 집중 리뷰
oaisstreamlit 리뷰 02_1st_server/pages/0_03_대리업체관리.py --focus session

# 캐싱 전략 집중 리뷰
oaisstreamlit 리뷰 02_1st_server/pages/0_03_대리업체관리.py --focus cache
```

**동작**:

1. 지정한 Streamlit 페이지 파일 읽기
2. streamlit-code-reviewer 에이전트 실행
3. 7가지 핵심 영역 검토 수행
4. 리뷰 보고서 출력

**검토 영역**:

| 영역 | 설명 |
|------|------|
| Session State | 초기화 패턴, 상태 일관성, 불필요한 상태 |
| 캐싱 전략 | @st.cache_data, @st.cache_resource, TTL 설정 |
| oais 모듈 통합 | oais.db, oais.date_utils 표준 사용 |
| 레이아웃 패턴 | st.columns, st.tabs, st.expander 활용 |
| 성능 최적화 | 불필요한 재실행 방지, 위젯 키 관리 |
| 에러 처리 | st.error/warning/success 피드백 |
| 보안 | SQL 인젝션 방지, 민감정보 노출 방지 |

**옵션**:

| 옵션 | 설명 |
|------|------|
| `--focus session` | Session State 집중 검토 |
| `--focus cache` | 캐싱 전략 집중 검토 |
| `--focus perf` | 성능 최적화 집중 검토 |
| `--focus layout` | 레이아웃/UX 집중 검토 |
| `--fix` | 발견된 문제 자동 수정 (확인 후) |
| `--lang en` | 영어 출력 (기본: 한국어) |

**실행 로직** (Claude가 따라야 할 단계):

```
1. 대상 파일 읽기:
   Read("[파일경로]")

2. streamlit-code-reviewer 에이전트 호출:
   Task(
     subagent_type="streamlit-code-reviewer",
     prompt="[파일경로] 파일을 Streamlit 전용 코드 리뷰 수행"
   )

3. 리뷰 보고서 출력:
   - 치명적 문제 (즉시 수정 필요)
   - Session State 이슈
   - 캐싱 최적화
   - oais 모듈 통합
   - 레이아웃/UX 개선
   - 잘된 점
   - 요약 및 품질 점수
```

**출력 형식**:

```

# Phase 1: 개요 (문서 준비)

개발서 파일을 생성하고, 페이지의 기본적인 개요(목적, 사용자, 주요 기능)를 정의합니다.

---

## 1.1 개요 워크플로우

```
1. oaisstreamlit 생성 명령어 실행
        ↓
2. oaisstreamlit 개요 명령어 실행
   - 목적 및 범위 정의
   - 주요 기능 목록화
```

---

# Phase 2: PRD (기획 상세)

PRD 문서(d0001_prd.md)를 바탕으로 해당 페이지의 상세 기획(화면, 기능, 데이터)을 확정합니다.

---

## 2.1 PRD 워크플로우

```
1. PRD 문서 (d0001_prd.md) 참조
        ↓
2. 화면 구성 및 기능 상세 정의
   - UI 영역별 컴포넌트 및 동작 정의
   - 입력 데이터 및 유효성 검사 규칙
        ↓
3. 템플릿 '2. PRD' 섹션 작성
```

---

## 2.2 PRD 서브에이전트 호출

```
Task(
  subagent_type="streamlit-page-planner",
  prompt="다음 페이지의 상세 PRD를 작성하라:

  **페이지**: [페이지명]
  **참조**: d0001_prd.md 해당 섹션 및 참조 이미지

  작성 항목 (템플릿 섹션 2):
  1. 화면 구성 (Wireframe description)
  2. 주요 기능 목록 및 로직
  3. 데이터 요구사항 (입력/출력)"
)
```

---

# Phase 3: 플랜 (구현 계획)

확정된 PRD를 기반으로 구체적인 구현 계획(WBS)과 실행 전략을 수립합니다.

---

## 3.1 플랜 워크플로우

```
1. 상세 PRD (섹션 2) 확인
        ↓
2. 기술적 구현 방안 검토
   - oais 모듈 활용 여부
   - 데이터베이스 구조 검토
        ↓
3. WBS 수립 (Phase/Task 분해)
        ↓
4. 템플릿 '3. 플랜' 섹션 작성
```

---

## 3.2 플랜 서브에이전트 호출

```
Task(
  subagent_type="streamlit-page-planner",
  prompt="다음 페이지의 구현 계획을 수립하라:

  **페이지**: [페이지명]
  **PRD**: 본 문서(d{번호}_{페이지}.md) 섹션 2 참조

  작성 항목 (템플릿 섹션 3):
  1. 기술적 접근법 (oais 모듈, DB)
  2. WBS (Phase/Task 목록)
  3. 상세 태스크 및 우선순위
  4. 리스크 관리"
)
```

---

## 3.3 플랜 출력물 형식

```markdown

### 3.1 기술적 접근법
- **프레임워크**: Streamlit, oais 모듈
- **주요 전략**: ...

### 3.2 WBS
| Phase | Task | 설명 | 우선순위 |
|-------|------|------|----------|
| P1 | UI 구조 | ... | High |
| P2 | 기능 구현 | ... | High |

### 3.3 상세 태스크
- **T1-1**: ...
```

# Phase 4: 설계 (UI/UX & DB)

페이지의 레이아웃, 컴포넌트 배치, 데이터베이스 구조를 설계합니다.

---

## 4.1 설계 워크플로우

```
1. 플랜 문서 확인
        ↓
2. 사용자 워크플로우 파악
        ↓
3. 정보 구조 설계 (IA)
        ↓
4. 와이어프레임 및 DB 스키마 설계
        ↓
5. Streamlit 컴포넌트 매핑
        ↓

        ↓

## 4.4 DB 설계 시 표준용어 적용

DB 컬럼/테이블 설계 시 `v/template/db_standard_word.md`의 표준용어를 준수합니다.

### 주요 카테고리별 표준 약어

| 카테고리 | 표준용어 예시 |
|----------|--------------|
| 식별자 | id, seq, cd, no, key |
| 명칭 | nm, ttl, lbl, desc, cont |
| 일시 | dt, tm, dttm, yr, mon, day |
| 상태 | stat, typ, cat, lvl, flg |
| 사용자 | usr, acct, pwd, email, tel |
| 조직 | comp, dept, team, org, cust |
| 금융 | amt, prc, tax, vat, pay |
| 파일 | file, doc, attch, img, path |
| 시스템 | sys, cfg, log, hist, sess |

### 자주 사용하는 컬럼명 패턴

| 패턴 | 설명 | 예시 |
|------|------|------|
| `*_id` | 고유 식별자 | user_id, task_id |
| `*_cd` | 코드 값 | status_cd, type_cd |
| `*_nm` | 이름/명칭 | user_nm, comp_nm |
| `*_dt` | 날짜 | created_dt, updated_dt |
| `*_yn` | 여부 (Y/N) | use_yn, del_yn |
| `*_no` | 번호 | seq_no, order_no |
| `*_cnt` | 개수 | view_cnt, like_cnt |
| `*_amt` | 금액 | total_amt, tax_amt |

### 표준용어 적용 원칙

1. **신규 테이블/컬럼 생성 시**: 반드시 표준용어집 참조
2. **약어 일관성**: 동일 의미는 동일 약어 사용 (예: 이름은 항상 `nm`)
3. **복합어 조합**: `[대상]_[속성]_[유형]` 순서 (예: `user_login_dt`)
4. **참조 문서**: `v/template/db_standard_word.md`, `doc/d0006_db.md`

## 5.3 페이지 기본 구조 템플릿

```python
import streamlit as st
from oais import ui, db, data_processing

# 페이지 설정
ui.setup_page_config(page_title="페이지명")

# 페이지 헤더
ui.show_page_header("페이지 제목", "페이지 설명")

# 세션 상태 초기화
if "selected_item" not in st.session_state:
    st.session_state.selected_item = None

# 사이드바
with st.sidebar:
    search_term = st.text_input("검색")
    filter_value = st.selectbox("상태", ["전체", "활성", "비활성"])

# 데이터 로드
data = db.get_data(search_term, filter_value)

# 메인 콘텐츠
st.dataframe(data)
```
