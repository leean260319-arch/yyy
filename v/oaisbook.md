## 1. 개요

**목적**: 도서 및 유튜브 영상에 대한 상세 서머리를 자동 생성하는 스킬입니다.

**핵심 기능**:
- 도서 서머리 생성 (7개 섹션, 약 50KB)
- 유튜브 서머리 생성 (8개 단계)
- 서머리 → Word 문서 변환
- 배치 처리 및 진행 상태 관리

**참조 문서**:
- `01_algorithm/ps9003_park_summary_v03.md` (마스터 문서)
- `01_algorithm/ps9003e_도서서머리생성.md` (도서 서머리 명령어)
- `01_algorithm/ps9003f_도서서머리워드문서생성.md` (워드 변환 명령어)
- `01_algorithm/ps9003c_스크립트.md` (스크립트 라이브러리)

## 2. 명령어 체계

### 2.1 기본 명령어

| 명령어 | 설명 | 역할 |
|--------|------|------|
| `oaisbook` | **도서 서머리 생성 (기본)** | 미완성 도서 순차 처리 |
| `oaisbook version` | 스킬 버전 정보 (v02) | 터미널 |

## 3. 도서 서머리 생성

### 3.1 입출력 경로

**입력**:
- Excel 파일: `data/ps1001_박기성/BOOKQAI.xlsx`
  - Q1 시트: 도서명 목록 (A열)
  - Q2 시트: 섹션 주제 (A열 - 7개 섹션)
- 샘플 파일: `data/ps1001_박기성/노화도 설계하는 시대가 온다.md`

**출력**:
- 출력 폴더: `data/ps9003/01_도서/`
- 파일 형식: `[도서명].md`
- 파일 크기: 약 50KB

### 3.2 7개 섹션 구조

| 섹션 | 제목 | 설명 |
|------|------|------|
| 1 | 각 장별로 핵심내용 정리 | 장별 30줄 내외, 핵심키워드별 3줄 내외 |
| 2 | 핵심 통찰과 울림을 주는 구절 30개 | 맥락/의미/원문 + 5분 낭독 스크립트 |
| 3 | 핵심키워드 정리 (30분 서사적 강의용) | 문제제기/원인분석/해결전략/삶의변화 |
| 4 | 강의안 작성 | 시간 배분, 서사적 전개, 청중 참여 |
| 5 | 강의전문 (Full Script) | 4,500~5,500자 분량 연설문 |
| 6 | 구글 슬라이드 작성 가이드 | 15-20장 분량 프레젠테이션 설계 |
| 7 | 플래시카드 및 퀴즈 | 플래시카드 20개, 퀴즈 18문항 |

### 3.3 실행 프로세스

```
Step 1: 진행 상태 확인
→ ps9003_park_summary_v03.md 섹션 4 "도서 서머리" 확인
→ 미완성 파일 목록 추출

Step 2: 첫 번째 미완성 도서 선택
→ data/ps9003/01_도서/[도서명].md

Step 3: 7개 섹션 순차 생성
→ 섹션 1 생성 → 저장 → 섹션 2 생성 → 저장 → ... → 섹션 7 생성 → 저장

Step 4: 진행 상태 업데이트
→ 완성본 +1, 미완성 -1

Step 5: 다음 도서로 자동 진행
→ 미완성 > 0 이면 Step 2로
→ 미완성 = 0 이면 완료
```

### 3.4 생성 지침

**원칙**:
- 원본 충실성: 책의 실제 내용, 저자의 원문에 충실
- 분량 기준: 각 섹션이 템플릿 가이드 분량 충족
- 전체 목표: 서머리 MD 파일 약 50KB

**검색 및 참조**:
- 인터넷 검색 적극 활용 (영어 자료 포함)
- 차트 생성 제외 (시간 소요)

## 5. Word 문서 변환

### 5.1 실행 명령어

```bash
oaisbook word
```

### 5.2 입출력 경로

- 입력: `data/ps9003/01_도서/*.md`
- 출력: `data/ps9003/02_워드/*.docx`

### 5.3 스타일 설정

| 항목 | 설정값 |
|------|--------|
| 용지 크기 | A4 |
| 여백 | 상하좌우 2.5cm |
| 한글 폰트 | 맑은 고딕 |
| 영문 폰트 | Calibri |
| 본문 크기 | 11pt |
| 줄간격 | 1.5줄 |

### 5.4 Markdown → Word 변환 규칙

| Markdown | Word 스타일 |
|----------|------------|
| `# 제목` | 제목 1 (18pt, 굵게) |
| `## 제목` | 제목 2 (16pt, 굵게) |
| `### 제목` | 제목 3 (14pt, 굵게) |
| `- 항목` | 글머리 기호 |
| `1. 항목` | 번호 매기기 |
| `**굵게**` | 굵게 |

from pathlib import Path

def find_project_root():
    """프로젝트 루트 자동 감지"""
    current = Path(__file__).resolve().parent if '__file__' in globals() else Path.cwd()
    while current != current.parent:
        if (current / "pyproject.toml").exists():
            return current
        current = current.parent
    return Path.cwd()

### 6.3 파일명 정리 함수

```python
def sanitize_filename(filename):
    """파일명에서 사용 불가능한 문자 제거"""
    invalid_chars = '<>:"/\\|?*'
    for char in invalid_chars:
        filename = filename.replace(char, '_')
    return ' '.join(filename.split()).strip()
```

## 7. AI 실행 규칙

### 7.1 필수 실행 원칙

- 명령어 인식 즉시 첫 번째 도서 섹션 1부터 시작
- 섹션 7까지 완료 → 자동으로 다음 도서
- 각 섹션 완료 후 파일 저장 → 다음 섹션 자동 진행
- 전체 서머리를 한 번에 생성하지 않고 **반드시 섹션별로 순차 생성**

### 7.2 절대 금지 사항

- "몇 권만 하시겠습니까?" 질문
- "시범적으로 3권만 해볼까요?" 제안
- 작업 범위 축소 제안
- 간략 버전 제시
- 옵션 선택지 제시

### 7.3 토큰 관리

- 전체 서머리 한 번에 생성 시 토큰 제한(32,000) 초과
- **반드시 섹션별로 순차 생성**
- 각 섹션 완료 후 파일에 저장
